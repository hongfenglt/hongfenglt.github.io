制作、更新自己的CocoaPods公有库

发表于 2018-09-13 | 分类于 iOS 开发

前言

CocoaPods 是为iOS/Mac开发而生的依赖管理工具。
开发 iOS 项目不可避免的要使用各种第三方开源库，而 CocoaPods 可以让我们很方便的管理第三方库，极大的提高我们的工作效率。
我们从事 iOS 开发一段时间后，肯定会整理一些自己常用的分类、封装一些 UI 组件、工具等等。这篇文章会带你从零开始创建属于你自己的 CocoaPods 库。当然库本身的代码内容需要你自己去创造。
关于 CocoaPods 具体的安装使用这里不再多说。

CocoaPods原理图

我下面画了一张简单的图方便理解 CocoaPods 工作原理。

如果理解了图上 CocoaPods 的流程，那么 CocoaPods 的制作流程就很明显了。

  1. 开发源码并上传到 Github 等代码托管平台
  2. 创建.spec文件并补全信息
  3. 上传.spec文件到远程索引库

2.3 打Tag标签
这个标签要跟.spec中的一致，因为最后 CocoaPods 会根据 .spec 文件中的 tag 去下载源码
例如 pod 'AFNetworking', '~> 3.0'
如果不指定版本默认会安装最新版

git tag 0.0.1
git push –tags

远程仓库 Releases 中就有了刚刚 push 上去的 tag ，如果打错 tag ，下面是删除 tag 的方法

git tag -d 0.0.1 // 删除本地 tag
git push origin :0.0.1 // 删除远程 tag

3.上传.spec文件到远程索引库

3.1 注册 Trunk
pod trunk register hongfengkt@gmail.com --verbose
注意上面邮箱换成你的真实邮箱，成功后你会收到一封邮件，需要点击验证
查看trunk pod trunk me
添加其他维护者(如果框架由多人维护)
pod trunk add-owner ARAnalytics kyle@cocoapods.org
更多详细内容查看官网：https://guides.cocoapods.org/making/getting-setup-with-trunk.html


以上只需操作一次

3.2 验证.spec文件
本地验证.spec： pod spec lint
如果没有问题会显示 LTTestLib.podspec passed validation.

如果有问题使用 pod spec lint --verbose 可以查看详细信息

通过以后提交到远程仓库

3.3 上传 sepc 到官方索引库
使用命令 pod trunk push LTTestLib.podspec

这条命令做了三件事：
验证远程 podspec 文件
上传 podspec 文件到 trunk 服务
将{project}.podspec文件转为{poject}.podspec.json文件

如果你的库依赖了其他库你可能需要加上 --use-libraries
如果你依赖的其他库有许多警告，可能会验证不通过，你需要加上 --allow-warnings
所以最后命令可能是

3.4 pod search LTTestLib 确认你的库可以被搜索到

如果这里搜索不到，是因为你本地是索引文件没有更新，参考最上面那张图。
删除 search_index.josn 在搜索就可以了

大功告成，这里能搜到就说明其他人也可以使用 pod 来安装你的库了。

你可以新建一个项目来测试一下，我们在 spec 文件中设置依赖 Reachability ，所以 Reachability 也会被安装

4.更新版本

更新你的框架，打新 tag
编辑 podspec 文件，更新 tag
重复3.2~3.4

之后在测试工程中更新你的库pod update LTTestLib
